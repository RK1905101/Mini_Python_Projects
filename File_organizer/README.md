### What Happens During Execution

The script performs the following operations:

1. **Scans** the target directory and all subdirectories recursively
2. **Categorizes** files based on their extensions (defined in `CONST.py`)
3. **Creates** organized folder structure: `Category/Year/Month/`
4. **Moves** files to their appropriate destinations
5. **Handles** duplicates by appending `(1)`, `(2)`, etc. to filenames
6. **Cleans up** empty directories left behind
7. **Logs** all operations to `file_organizer.log` in the target directory

> **Note:** Already organized files (inside category folders) are automatically skipped to prevent re-organization.

## üßë‚Äçüíª **Technical Implementation**

### Core Technologies

- **`os.scandir()`** ‚Üí High-performance recursive directory scanning with cached file attributes
- **`shutil.move()`** ‚Üí Safe file relocation with automatic cross-filesystem support (handles moves across different drives/mount points)
- **`os.walk()`** ‚Üí Bottom-up directory traversal for empty directory cleanup
- **Path objects** ‚Üí Type-safe file path handling with pathlib

### Smart Features

- **Intelligent duplicate handling** ‚Üí Checks destination existence before moving and auto-renames conflicts (e.g., `file(1).ext`, `file(2).ext`)
- **Category folder skipping** ‚Üí Prevents re-organizing already sorted files by skipping category directories during scan
- **Empty directory cleanup** ‚Üí Recursively removes empty folders after organizing (bottom-up traversal to handle nested empties)
- **Permission-aware scanning** ‚Üí Gracefully skips directories without read permissions
- **Comprehensive logging** ‚Üí Tracks all operations, errors, and exceptions in `file_organizer.log`
- **Date-based organization** ‚Üí Uses file creation time to organize into Year/Month subdirectories

### Why This Approach?

**`os.scandir()` for scanning:**
- ‚ö° **~2-3x faster** than `os.listdir()` or traditional `os.walk()` for large directories
- üìä Caches file stat information (type, size, timestamps) reducing redundant system calls
- üéØ Returns `DirEntry` objects with efficient `is_file()` and `is_dir()` methods

**`os.walk()` for cleanup:**
- üîÑ Bottom-up traversal (`topdown=False`) ensures child directories are processed before parents
- ‚úÖ Guarantees complete removal of nested empty directory structures
- üõ°Ô∏è Safe iteration while modifying directory tree

**`shutil.move()` for file operations:**
- üîÑ Automatically handles cross-filesystem moves (copy + delete when needed)
- üõ°Ô∏è More robust than `Path.rename()` which fails across mount points
- ‚úÖ Works seamlessly with both Path objects and strings
